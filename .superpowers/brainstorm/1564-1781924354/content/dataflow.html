核心数据流设计

上传发布流程是最复杂的链路，涉及预签名 URL → 直传 MinIO → 关联业务数据

发布笔记完整流程（关键路径）
Step 1: 创建草稿
POST /api/note/draft  { userId:1001, title:"标题", content:"正文" }
  → gateway:8080 → note-service → leaf-service (生成noteId) → Cassandra
  ← { code:200, data:{ noteId:1234567890, status:"DRAFT" } }

Step 2: 获取预签名上传URL
GET /api/upload/presigned?fileName=cover.jpg&contentType=image/jpeg
  → gateway:8080 (Sentinel限流) → upload-service → MinIO
  ← { code:200, data:{ uploadUrl:"http://minio:9000/...", objectKey:"2026/06/20/uuid_cover.jpg", expiresAt:... } }

Step 3: 直传MinIO (浏览器 → MinIO，不经过后端!)
PUT {uploadUrl}  ← 原始图片二进制
  → MinIO:9000 (直接上传，5分钟内有效)
  ← 200 OK

Step 4: 发布笔记（关联图片 + 状态变更）
POST /api/note/publish  { noteId:1234567890, fileName:"cover.jpg", contentType:"image/jpeg" }
  → gateway:8080 → note-service → Cassandra (UPDATE status="PUBLISHED", objectKey=...)
  ← { code:200, data:{ noteId:..., status:"PUBLISHED", uploadUrl:... } }
浏览 Feed 流程
GET /api/note/detail?noteId=ID
  ← note + comments
