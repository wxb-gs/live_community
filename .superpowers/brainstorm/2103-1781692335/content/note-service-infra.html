基础设施集成与工程质量

美团 Leaf 分布式 ID 集成

两种模式分工
模式 用途 原理 性能
号段 (Segment) 笔记 ID DB 批量取号，本地缓存，用完再取 极高，无网络调用
雪花 (Snowflake) 评论 ID workerId + 时间戳 + 序列号，本地生成 极高，完全本地

Leaf Server 独立部署，note-service 通过 HTTP 调用获取号段。Snowflake 通过 ZooKeeper 分配 workerId。

错误处理策略

Gateway NoteController 异常处理：

  draftId 不存在          → Result.error(404, "草稿不存在")
  发布时 draftId 非本人    → Result.error(403, "无权操作")
  重复发布                 → Result.error(409, "笔记已发布")
  Leaf ID 生成失败         → Result.error(503, "服务繁忙，请重试")
  Cassandra 超时           → Result.error(500, "评论加载失败"), 笔记本体仍返回
  MinIO 上传失败           → Result.error(500, "发布失败，请重试"), 事务回滚

降级策略：
  - 笔记详情接口：Cassandra 超时时 comments 字段返回空列表，不阻塞笔记数据
  - 笔记列表接口：不查 Cassandra，count 字段用 MySQL 冗余值
