笔记服务 — 架构方案对比

现有模块：common · gateway · upload-service（Dubbo RPC + MinIO）

A

新增 note-service 模块（推荐）

新建独立 Dubbo 服务，复用 upload-service 做文件存储，保持职责单一

B

扩展 upload-service 为内容服务

把笔记、草稿、评论逻辑都加到 upload-service，一个服务搞定所有内容

C

note-service 用 REST API

新建独立模块，Gateway 通过 HTTP 路由转发，不走 Dubbo RPC

模块依赖关系

方案 A：新增 note-service（Dubbo RPC）
  Client
    │  HTTP
    ▼

    │  Dubbo Triple          │  Dubbo Triple
    ▼                        ▼

│  note-service    │  │  upload-service      │

方案 B：扩展 upload-service
  Client
    │  HTTP
    ▼
┌──────────────────────────────────────┐
│  gateway (8080)                      │
│  @DubboReference UploadRpcService    │
└──────────────────────────────────────┘
    │  Dubbo Triple (所有接口)
    ▼
┌──────────────────────────────────────┐
│  upload-service (内容单体)            │

方案 C：note-service REST API
  Client
    │  HTTP
    ▼

    ▼                          ▼
┌──────────────────┐  ┌──────────────────────┐
│  note-service    │  │  upload-service      │
