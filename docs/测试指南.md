# 测试指南

## 1. 测试金字塔

```
         ┌─────────┐
         │   E2E   │  少量，全链路，最慢（分钟级）
         │  端到端  │
        ┌┴─────────┴┐
        │ Integration│  适中，跨模块，中速（秒级）
        │   集成测试  │
       ┌┴───────────┴┐
       │    Unit     │  大量，单方法，最快（毫秒级）
       │   单元测试   │
       └────────────┘
```

| 层 | 占比 | 速度 | 目的 | 本项目中 |
|----|------|------|------|---------|
| **Unit** | 70% | 毫秒 | 验证单个方法的逻辑正确性 | Service 层纯逻辑 |
| **Integration** | 20% | 秒 | 验证模块间交互、数据库、RPC | Controller + DB + Dubbo |
| **E2E** | 10% | 分钟 | 验证完整用户场景 | HTTP 全链路 |

## 2. 单元测试

单元测试只测**一个方法**的纯逻辑，不启动 Spring 容器，不连数据库。所有外部依赖用 Mockito Mock。

### 2.1 基础示例

```java
package com.example.note.service;

import static org.junit.jupiter.api.Assertions.*;
import static org.mockito.ArgumentMatchers.*;
import static org.mockito.Mockito.*;

import com.example.note.entity.NoteRow;
import com.example.note.repository.NoteMysqlRepository;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.junit.jupiter.MockitoExtension;

import java.util.Optional;

@ExtendWith(MockitoExtension.class)
class NoteServiceTest {

    @Mock
    private NoteMysqlRepository noteMysqlRepository;
    @Mock
    private CommentRepository commentRepository;
    @Mock
    private MinioClient minioClient;
    @Mock
    private LeafRpcService leafRpcService;
    @Mock
    private NoteSyncProducer noteSyncProducer;

    @InjectMocks
    private NoteService noteService;

    // ── 正向测试 ──

    @Test
    void createDraft_shouldReturnDraftResponse_whenValidRequest() {
        // Given
        CreateDraftRequest req = new CreateDraftRequest();
        req.setUserId(1001L);
        req.setTitle("测试标题");
        req.setContent("测试内容");

        IdResponse idResp = new IdResponse();
        idResp.setId(42L);
        when(leafRpcService.generateSegmentId("note")).thenReturn(idResp);
        when(noteMysqlRepository.findById(42L)).thenReturn(Optional.of(mock(NoteRow.class)));

        // When
        CreateDraftResponse resp = noteService.createDraft(req);

        // Then
        assertNotNull(resp);
        assertEquals(42L, resp.getNoteId());
        assertEquals("DRAFT", resp.getStatus());
        verify(noteMysqlRepository).upsert(eq(42L), eq(1001L), eq("测试标题"),
                eq("测试内容"), anyString(), isNull(), isNull(), isNull(),
                eq("DRAFT"), anyLong(), anyLong());
    }

    // ── 异常测试 ──

    @Test
    void publishNote_shouldThrowException_whenNoteNotFound() {
        PublishNoteRequest req = new PublishNoteRequest();
        req.setNoteId(999L);
        when(noteMysqlRepository.findById(999L)).thenReturn(Optional.empty());

        assertThrows(RuntimeException.class, () -> noteService.publishNote(req));
    }

    // ── 边界测试 ──

    @Test
    void extractSummary_shouldTruncate_whenContentExceeds200Chars() {
        String longContent = "A".repeat(300);
        // 通过 createDraft 间接测试 extractSummary
        // ...
    }
}
```

### 2.2 Mockito 关键操作

| 操作 | 代码 |
|------|------|
| Mock 返回值 | `when(repo.findById(1L)).thenReturn(Optional.of(row))` |
| Mock 抛异常 | `when(repo.save(any())).thenThrow(new RuntimeException())` |
| 验证调用 | `verify(repo).save(any())` |
| 验证次数 | `verify(repo, times(2)).save(any())` |
| 验证未调用 | `verify(repo, never()).delete(any())` |
| 参数捕获 | `ArgumentCaptor<NoteEntity> captor = ...` |

### 2.3 什么值得单元测试

| 优先级 | 示例 |
|--------|------|
| 🔴 高 | `extractSummary()` 截断逻辑、`generateObjectKey()` 格式化 |
| 🔴 高 | `toggle()` 点赞/取消状态切换 |
| 🟡 中 | `toDetailResponse()` 字段映射 |
| 🟢 低 | 纯 getter/setter、简单委派 |

### 2.4 什么不值得单元测试

- 纯数据对象（DTO、Entity）
- 只有一行委托的 Controller 方法
- Spring 配置类
- 需要启动容器的场景 → 用集成测试

## 3. 集成测试

集成测试启动 Spring 上下文，验证真实组件协作。适合测 Controller、Repository、Dubbo RPC。

### 3.1 Controller 层（MockMvc）

本项目已有示例 `upload-service/.../UploadControllerTest.java`：

```java
@SpringBootTest(classes = UploadServiceApplication.class,
    properties = {
        "spring.cloud.nacos.discovery.enabled=false",
        "dubbo.registry.address=N/A",
        "minio.endpoint=http://localhost:9000",
        "minio.access-key=minioadmin",
        "minio.secret-key=minioadmin",
        "minio.bucket=uploads"
    })
@AutoConfigureMockMvc
class UploadControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    void presignedUrl_shouldReturn400_whenMissingParams() throws Exception {
        mockMvc.perform(get("/api/upload/presigned"))
                .andExpect(status().isBadRequest());
    }

    @Test
    void presignedUrl_shouldReturn200_whenValidParams() throws Exception {
        mockMvc.perform(get("/api/upload/presigned")
                        .param("fileName", "test.jpg")
                        .param("contentType", "image/jpeg"))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.data.uploadUrl").isNotEmpty())
                .andExpect(jsonPath("$.data.objectKey").isNotEmpty());
    }
}
```

**关键注解说明**：

| 注解/参数 | 作用 |
|-----------|------|
| `@SpringBootTest` | 启动完整 Spring 上下文 |
| `classes = XxxApplication.class` | 指定启动类（避免扫到不需要的 Bean） |
| `properties = {...}` | 覆盖配置（禁用 Nacos/Dubbo 避免连外部） |
| `@AutoConfigureMockMvc` | 注入 `MockMvc`，不启动真实 HTTP Server |

### 3.2 Repository 层（DB 集成）

需要真实数据库时用 `@DataJdbcTest` 或 `@SpringBootTest`（本项目用 JdbcTemplate）：

```java
@SpringBootTest(classes = NoteServiceApplication.class,
    properties = {
        "spring.cloud.nacos.discovery.enabled=false",
        "dubbo.registry.address=N/A",
        "spring.cassandra.contact-points=localhost",
        "spring.cassandra.port=19042",
        "spring.datasource.url=jdbc:mysql://localhost:13306/live_community",
        "spring.kafka.bootstrap-servers=localhost:9092"
    })
class NoteMysqlRepositoryTest {

    @Autowired
    private NoteMysqlRepository repo;

    @Test
    void upsertAndFindById_shouldReturnSavedNote() {
        long noteId = System.currentTimeMillis();  // 避免主键冲突
        repo.upsert(noteId, 1L, "测试", "内容", "摘要",
                null, null, null, "DRAFT", noteId, noteId);

        Optional<NoteRow> result = repo.findById(noteId);
        assertTrue(result.isPresent());
        assertEquals("测试", result.get().getTitle());
        assertEquals("DRAFT", result.get().getStatus());

        // 清理
        repo.delete(noteId);
    }
}
```

## 4. Testcontainers — 无需预装 Docker 容器的集成测试

不需要手动启动 Docker Compose，Testcontainers 自动管理容器生命周期。

### 4.1 依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-testcontainers</artifactId>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.testcontainers</groupId>
    <artifactId>mysql</artifactId>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.testcontainers</groupId>
    <artifactId>cassandra</artifactId>
    <scope>test</scope>
</dependency>
```

### 4.2 使用

```java
@SpringBootTest
@Testcontainers
class NoteMysqlRepositoryTest {

    @Container
    static MySQLContainer<?> mysql = new MySQLContainer<>("mysql:8.0")
            .withDatabaseName("live_community")
            .withUsername("root")
            .withPassword("root");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", mysql::getJdbcUrl);
        registry.add("spring.datasource.username", mysql::getUsername);
        registry.add("spring.datasource.password", mysql::getPassword);
    }

    @Autowired
    private NoteMysqlRepository repo;

    @Test
    void shouldInsertAndQuery() {
        // 不需要自己启动 MySQL Docker
        repo.upsert(1L, 1L, "标题", "内容", "摘要",
                null, null, null, "DRAFT", 0L, 0L);
        assertTrue(repo.findById(1L).isPresent());
    }
}
```

## 5. 性能测试

### 5.1 JMH（Java Microbenchmark Harness）

测单个方法的纳秒/微秒级性能。适合测 ID 生成器、序列化、字符串处理。

```java
@BenchmarkMode(Mode.Throughput)
@OutputTimeUnit(TimeUnit.SECONDS)
@State(Scope.Benchmark)
public class ExtractSummaryBenchmark {

    private String longContent;

    @Setup
    public void setup() {
        longContent = "A".repeat(10000);
    }

    @Benchmark
    public String extractSummary() {
        if (longContent.length() > 200) {
            return longContent.substring(0, 200) + "...";
        }
        return longContent;
    }
}
```

### 5.2 接口压测（wrk / JMeter）

针对 HTTP 接口做 QPS 压测。

```bash
# wrk 压测搜索接口
wrk -t4 -c100 -d30s --latency http://localhost:8080/api/search?keyword=微服务

# 结果示例:
#  Requests/sec:   3200.15
#  Latency avg:    31.20ms
#  Latency p99:    85.00ms
```

**本项目中需要压测的接口**：

| 接口 | 原因 |
|------|------|
| `GET /api/search?keyword=` | 搜索是高频接口，需要知道 p99 延迟 |
| `POST /api/interaction/toggle` | 点赞接口，热点笔记下的并发 |
| `GET /api/note/detail?noteId=` | 详情页最常访问 |
| `GET /api/upload/presigned` | 上传前获取预签名 URL |

### 5.3 数据库查询性能

用 `EXPLAIN` 检查 MySQL 查询计划：

```sql
EXPLAIN SELECT * FROM note WHERE status = 'PUBLISHED' ORDER BY created_at DESC LIMIT 20;
-- 关注 type 列: ALL(全表) < index < range < ref < const
-- 关注 rows 列: 扫描行数，越少越好
```

### 5.4 性能测试时机

```
❌ 不测: 每个 PR 都跑 JMH → 浪费时间
✅ 应该测:
  - 新算法/新数据结构上线前
  - 预期 QPS > 1000 的接口
  - 发现慢查询时
  - 技术选型对比（如 XXL-JOB vs SchedulerLock）
```

## 6. 本项目测试现状与补全计划

### 现状

| 模块 | 单元测试 | 集成测试 | 状态 |
|------|---------|---------|------|
| `common` | 0 | 0 | ❌ 无测试 |
| `upload-service` | 0 | 1（UploadController） | 🟡 仅 1 个 |
| `gateway` | 0 | 0 | ❌ 无测试 |
| `leaf-service` | 0 | 0 | ❌ 无测试 |
| `note-service` | 0 | 0 | ❌ 无测试 |
| `auth-service` | 0 | 0 | ❌ 无测试 |
| `search-service` | 0 | 0 | ❌ 无测试 |
| `search-sync-service` | 0 | 0 | ❌ 无测试 |

### 优先补全顺序

```
1. note-service  NoteService       ← 业务核心，改动最频繁
2. note-service  NoteMysqlRepository ← 数据库交互验证
3. leaf-service   SegmentIdGenerator  ← ID 号段分配，错一个全链路崩
4. gateway        UploadController    ← Sentinel 限流逻辑需验证
5. auth-service   AuthController      ← 登录/注册，安全关键
6. search-service NoteSearchService   ← Function Score 查询结构验证
```

### 每个模块建议覆盖的方法

```
note-service:
  ☐ NoteService.createDraft()          — 单元测试（Mock Repository + RPC）
  ☐ NoteService.publishNote()          — 单元测试
  ☐ NoteService.getNoteDetail()        — 单元测试
  ☐ NoteMysqlRepository.upsert/findById — 集成测试（需要 MySQL）
  ☐ CommentLikeService.toggleAsync()   — 单元测试（Mock Cassandra）
  ☐ InteractionService.toggle()        — 单元测试（Mock Redis + Kafka）

leaf-service:
  ☐ SegmentIdGenerator.getId()         — 集成测试（需要 MySQL leaf_alloc 表）
  ☐ SnowflakeIdGenerator.nextId()      — 单元测试（纯计算，无外部依赖）

gateway:
  ☐ UploadController.presignedUrl()    — 集成测试（Mock Dubbo Reference）
  ☐ Sentinel fallback 逻辑             — 单元测试

auth-service:
  ☐ AuthController.login()             — 集成测试（Mock Dubbo）
  ☐ JWT token 生成/校验                — 单元测试

search-service:
  ☐ NoteSearchService.search()         — 集成测试（需要 ES）
  ☐ Function Score DSL 结构            — 单元测试（构造查询 → 对比 JSON）
```

## 7. Maven 测试命令速查

```bash
# 全量测试
mvn test

# 单模块测试
mvn test -pl note-service

# 单个测试类
mvn test -pl upload-service -Dtest=UploadControllerTest

# 单个测试方法
mvn test -pl note-service -Dtest=NoteServiceTest#createDraft_shouldReturnDraftResponse

# 跳过测试
mvn clean package -DskipTests

# 并行测试（多模块同时跑）
mvn test -T 4

# 生成覆盖率报告（需 jacoco 插件）
mvn test jacoco:report
```
