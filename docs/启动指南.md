# Spring Cloud 微服务系统 — 单机启动指南

## 系统概览

| 组件 | 类型 | 端口 | 说明 |
|------|------|------|------|
| Nacos Server | 中间件 | 8848 | 服务注册中心 + Sentinel 规则持久化 |
| Sentinel Dashboard | 中间件 | 自定义 | 限流规则可视化管控台（可选） |
| MinIO Server | 中间件 | 9000 | 对象存储，前端直传目标 |
| Gateway | 微服务 | 8080 | API 网关，含 Sentinel 限流 |
| upload-service | 微服务 | 8081 | 文件上传服务，生成 MinIO 预签名 URL |

**技术栈**：Spring Boot 3.2.6 + Spring Cloud 2023.0.3 + Spring Cloud Alibaba 2023.0.1.0 + MinIO 8.5.10 + Java 17

---

## 一、环境准备

### 1.1 基础依赖

- **JDK 17**（推荐 Eclipse Temurin / Amazon Corretto）
- **Maven 3.8+**
- **Nacos 2.x**（[GitHub Release](https://github.com/alibaba/nacos/releases) 下载 `nacos-server-2.x.x.zip`）
- **Sentinel Dashboard**（[GitHub Release](https://github.com/alibaba/Sentinel/releases) 下载 `sentinel-dashboard-1.8.x.jar`）
- **MinIO Server**（[官网下载](https://min.io/download) 或 `choco install minio` / `brew install minio`）

### 1.2 端口检查

启动前确保以下端口未被占用：

```
8848  — Nacos
9000  — MinIO API
9001  — MinIO Console（Web 管理界面）
8080  — Gateway
8081  — upload-service
```

Windows 检查命令：
```bash
netstat -ano | findstr "8848 9000 9001 8080 8081"
```

## 二、启动中间件

### 2.1 启动 Nacos

**Windows（PowerShell）：**
```powershell
cd C:\nacos\bin
.\startup.cmd -m standalone
```

**Linux / macOS：**
```bash
cd ~/nacos/bin
sh startup.sh -m standalone
```

> 必须使用 `standalone` 模式（单机模式），否则 Nacos 会尝试集群部署。

验证：浏览器打开 http://localhost:8848/nacos ，默认用户名密码均为 `nacos`。

### 2.2 启动 MinIO

**Windows：**
```powershell
minio server C:\minio-data --console-address :9001
```

**Linux / macOS：**
```bash
minio server ~/minio-data --console-address :9001
```

> 首次运行时 `C:\minio-data` 目录会自动创建。

验证：浏览器打开 http://localhost:9001 ，默认用户名密码均为 `minioadmin`。

登录后创建一个名为 `uploads` 的 bucket（与 `application.yml` 中 `minio.bucket` 配置一致）：

1. 点击 "Create Bucket"
2. Bucket Name 填写 `uploads`
3. 点击 "Create Bucket"

### 2.3 启动 Sentinel Dashboard（可选）

```bash
java -jar sentinel-dashboard-1.8.8.jar --server.port=8088
```

验证：浏览器打开 http://localhost:8088 ，默认用户名密码均为 `sentinel`。

> Sentinel Dashboard 用于实时监控和调整限流规则。不启动不影响网关限流——限流规则已存储在 Nacos 中。

## 三、配置 Nacos 限流规则

网关的 Sentinel 限流规则从 Nacos 动态读取，必须先创建此配置。

1. 打开 Nacos 控制台 http://localhost:8848/nacos
2. 左侧菜单 → **配置管理** → **配置列表**
3. 点击 **+** 新建配置，填写：

| 字段 | 值 |
|------|-----|
| Data ID | `gateway-flow-rules` |
| Group | `DEFAULT_GROUP` |
| 配置格式 | JSON |
| 配置内容 | 见下方 |

```json
[
  {
    "resource": "upload-service",
    "resourceMode": 0,
    "grade": 1,
    "count": 50,
    "intervalSec": 1,
    "controlBehavior": 0,
    "burst": 0,
    "maxQueueingTimeoutMs": 500
  }
]
```

> 含义：upload-service 路由每秒最多允许 50 个请求（QPS 模式），超出返回 HTTP 429。

点击 **发布**。

## 五、启动微服务

### 5.1 启动顺序

```
Nacos  →  MinIO  →  upload-service  →  gateway
```

> upload-service 必须先于 gateway 启动，确保 gateway 通过 Nacos 负载均衡发现它时服务已就绪。

### 5.2 启动 upload-service（端口 8081）

**Windows（PowerShell）：**
```powershell
cd C:\lib\codes\java_projects\spring_cloud_test\upload-service
Start-Process -NoNewWindow java -ArgumentList "-jar", "target/upload-service-1.0.0-SNAPSHOT.jar"
```

**Linux / macOS：**
```bash
cd ~/spring_cloud_test/upload-service
java -jar target/upload-service-1.0.0-SNAPSHOT.jar &
```

验证：打开 Nacos 控制台 → 服务管理 → 服务列表，应能看到 `upload-service` 已注册。

### 5.3 启动 Gateway（端口 8080）

**Windows（PowerShell）：**
```powershell
cd C:\lib\codes\java_projects\spring_cloud_test\gateway
Start-Process -NoNewWindow java -ArgumentList "-jar", "target/gateway-1.0.0-SNAPSHOT.jar"
```

**Linux / macOS：**
```bash
cd ~/spring_cloud_test/gateway
java -jar target/gateway-1.0.0-SNAPSHOT.jar &
```

验证：Nacos 服务列表中应同时出现 `gateway` 和 `upload-service`。

## 六、验证系统

### 6.1 验证服务注册

浏览器打开 http://localhost:8848/nacos → 服务管理 → 服务列表，应看到两个服务：

- `gateway` — 健康实例数 1
- `upload-service` — 健康实例数 1

### 6.2 验证上传接口

通过网关访问 upload-service 的预签名 URL 接口：

```bash
curl "http://localhost:8080/api/upload/presigned?fileName=test.txt&contentType=text/plain"
```

正常响应示例：
```json
{
  "code": 200,
  "msg": null,
  "data": {
    "uploadUrl": "http://localhost:9000/uploads/2026/06/12/abc123_test.txt?...",
    "objectKey": "2026/06/12/abc123_test.txt",
    "expiresAt": 1718150400
  }
}
```

### 6.3 验证限流

用压测工具（如 `ab`、`wrk`、`hey`）发送超过 50 QPS 的请求，应能收到 HTTP 429 响应：

```json
{"code":429,"msg":"Too many requests, please try again later"}
```

## 七、同时运行（单机多终端）

由于四个组件（Nacos、MinIO、upload-service、gateway）都需要独立进程，推荐以下方式同时运行：

### 方案 A：多终端窗口

开 4 个终端窗口，每个窗口启动一个组件：

| 窗口 | 启动内容 |
|------|----------|
| 窗口 1 | Nacos |
| 窗口 2 | MinIO |
| 窗口 3 | `java -jar upload-service/target/upload-service-1.0.0-SNAPSHOT.jar` |
| 窗口 4 | `java -jar gateway/target/gateway-1.0.0-SNAPSHOT.jar` |

### 方案 B：IDE 内启动（推荐开发时使用）

在 IntelliJ IDEA 中：
1. 为 `GatewayApplication` 和 `UploadServiceApplication` 分别创建 Run Configuration
2. 勾选 "Allow multiple instances"
3. 先启动 `UploadServiceApplication`，再启动 `GatewayApplication`

### 方案 C：后台进程（Linux / macOS）

```bash
# 启动 upload-service（打开一个终端窗口）
java -jar upload-service/target/upload-service-1.0.0-SNAPSHOT.jar

# 启动 gateway（打开另一个终端窗口）
java -jar gateway/target/gateway-1.0.0-SNAPSHOT.jar
```

## 八、常见问题

### Q1：启动时提示 "Address already in use"

端口被占用。用 `netstat -ano | findstr <端口>`（Windows）或 `lsof -i :<端口>`（macOS/Linux）查找占用进程并关闭。

### Q2：upload-service 注册不上 Nacos

检查 `application.yml` 中 `spring.cloud.nacos.discovery.server-addr` 是否正确指向 `localhost:8848`。如果 Nacos 装在远程机器，需要修改此处地址。

### Q3：网关路由 503 Service Unavailable

Nacos 中 upload-service 已注册，但网关报 503。可能原因：
- upload-service 刚启动，Nacos 摘除不健康实例有延迟（默认 15s）
- 确认 `spring.application.name` 与网关路由中的 `lb://upload-service` 完全一致（包括大小写）

### Q4：Sentinel 规则不生效

确认 Nacos 中已创建 `gateway-flow-rules` 配置。检查配置内容 JSON 格式是否正确（末尾不能有多余逗号）。

### Q5：MinIO 连接失败

upload-service 启动日志中若看到 `Connection refused: localhost:9000`，确认 MinIO 已启动，且 `application.yml` 中 `minio.endpoint` 正确。

## 九、停止系统

按与启动相反的顺序关闭：

1. 停 Gateway：终端中 `Ctrl+C`
2. 停 upload-service：终端中 `Ctrl+C`
3. 停 MinIO：终端中 `Ctrl+C`
4. 停 Nacos：进入 Nacos `bin/` 目录执行 `shutdown.cmd` 或 `shutdown.sh`
5. 停 Sentinel Dashboard：终端中 `Ctrl+C`
